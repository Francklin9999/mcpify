# PRODUCT.md - MCP Forge

register: product

## Product purpose
MCP Forge turns any website into a runnable MCP (Model Context Protocol) server. Paste a URL -> it captures
the page's real network traffic and content -> generates action-capable tools -> you download a server and
run it locally so an LLM (Claude, etc.) can act on that site. A library of curated + auto-generated servers,
a self-healing monitor that keeps them alive, and a Chrome chat extension that generates one from any page.

## Users
Developers and AI builders who want to give an LLM real capabilities on real websites without hand-writing
an integration. They value: it actually works, runs locally (their machine, their creds), and is fast to try.

## Brand / tone
Confident, technical, a little playful. "Generate, don't integrate." Show the working thing, not hype.
We are a builder's tool - concrete verbs, real artifacts (server.ts, a config snippet), live status.

## Visual direction (user-specified)
Inspired by Jobright (jobright.ai): clean, bright, **white + green**, rounded cards, generous whitespace,
a scannable list of scored items (their "match %" -> our **confidence score** band: color = health).
Must have a **landing page** (marketing hero) and a **dark/night mode**. Green is the brand accent.

## Anti-references
Not dark-terminal "hacker" aesthetic. Not SaaS-cream + serif editorial. Not purple gradients. Not a wall of
identical icon-heading-text cards. The confidence band is the one bold, functional color moment.

## Strategic principles
- The confidence score (health = color) is the product's signature visual - make it prominent, like Jobright's match %.
- Every screen shows something real: a generated tool, a live job status, a runnable file.
- Light is the default (bright, approachable, Jobright-like); dark mode is a first-class equal, not an afterthought.
